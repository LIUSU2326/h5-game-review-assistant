# 本地工具 UI 与 EXE 打包说明

## 当前形态

本项目现在有三种启动方式：

| 方式 | 命令或路径 | 用途 |
|---|---|---|
| 本地网页工具 | `npm.cmd run app` 后打开 `http://127.0.0.1:4177` | 日常开发和调试 |
| Electron 桌面开发模式 | `npm.cmd run desktop` | 验证桌面外壳 |
| Windows exe 候选包 | `dist\H5游戏评测助手 1.1.0-rc.1.exe` | v1.1 RC1 单文件 portable 候选 |
| Windows 文件夹版 | `dist\win-unpacked\H5游戏评测助手.exe` | 当前更适合反复测试 |

## 已验证能力

- 本地网页 UI 可以读取 Gemini 和飞书配置状态。
- UI 可以触发单款游戏 pipeline。
- UI 已加入首次配置向导，会按状态提示 Gemini、飞书凭证、多维表格、字段检查、标签同步和队列预演是否完成。
- UI 已加入运行档位选择：`poc_smoke` 用于极短烟测，`poc_review` 用于 60 秒 POC，`full_30min` 用于正式 30 分钟总采样。
- UI 已加入批量队列：可填入多个 `game_id`，先预演队列，再按当前运行档位逐款执行。
- 批量队列会写入 `batch/dry_run.json` 和 `batch/last_run.json`，用于查看计划、执行结果、失败项和每款游戏的退出码。
- 批量队列 UI 已显示每款任务状态、耗时和失败尾巴，并提供“填入失败项”按钮，用于失败后只重跑失败游戏。
- 批量队列已支持任务级自动重试和归档。每次真实执行会在 `batch/runs/<时间>-execute-<档位>/` 生成 `run.json`，并为每次尝试保存 stdout/stderr 日志；`poc_review` 默认单款失败后整款重试 1 次，`full_30min` 默认不自动重跑整款，避免额外消耗 30 分钟。
- UI 已加入结果总览：可以按全部、已完成、需复核、失败/缺失、证据完整筛选游戏，并在详情页触发“重跑采集 / 重跑 AI / 重写飞书”。
- UI 已加入质量提示：自动标出采集未完整、目标游戏不匹配、截图不足、广告/弹窗遮挡、低置信度字段、标签需复核和飞书未写入。
- UI 已加入人工复核标记：每款游戏详情页可以本地保存“待复核 / 已通过 / 需修改”和复核备注，记录会写入 `evidence/<game_id>/review_status.json`；飞书配置完整时会同步到 `Review Status`、`Review Notes`、`Review Updated At`。
- 人工复核区已支持“填入质量提示”，可以把当前质量风险带入备注框，再由使用者决定是否保存和同步飞书。
- UI 已完成 v1.0 RC1 工作台收口：侧栏、顶部状态、配置向导、批量队列、结果表格、质量提示、截图证据和人工复核区都按产品工作台风格重新压实间距、层级和状态色。
- 证据统一存放在数据目录的 `evidence/<game_id>/`：每款游戏自动创建独立文件夹，截图、网络摘要、trace、视频和报告都放在该目录下。
- 配置页已加入“本地证据存储”说明，会直接显示当前真实数据目录、每款游戏的证据文件夹规则、截图上传策略和视频保存策略。
- 运行档位已支持本地短视频证据：`poc_review` 默认录 1 段 15 秒 `.webm`，`full_30min` 默认录 1 段 30 秒 `.webm`，视频只保存在本地，不上传飞书。
- 飞书配置里已加入“写入飞书时上传截图附件”开关。开启后，真实写入飞书时会把截图上传到 `Screenshot Attachments` 附件字段；关闭时只写本地截图路径。
- 正式 30 分钟采样已改成每个设备/网络档位独立浏览器进程，并使用低频轻触策略，避免长时间高频点击导致页面或浏览器异常。`full_30min` 是单款游戏总计 30 分钟，当前会分摊到 4 个设备/网络档位。
- 正式档会启用部分失败检查：任一设备档位采集失败时，任务会真实标记失败，避免误以为已经完成。
- UI 可以触发快速检查、字段检查、标签库同步、批量计划刷新。
- Electron 文件夹版可以启动桌面窗口并拉起内置本地服务。
- 打包产物中未包含本机 `.env`、Gemini API Key、`config/feishu.local.json` 或 Feishu App Secret。
- Windows 打包请使用 `npm run dist:win`。该命令会把 Electron 临时目录和构建缓存放到项目内的 `.build-temp/`、`.build-cache/`，成功后自动清理临时目录，避免反复打包占满 C 盘。
- 桌面版已拆分程序目录和用户数据目录。打包后配置、样本、证据、日志会初始化到独立数据目录，而不是写进程序安装目录。
- 打包后的用户数据目录会自动初始化 `config/run_profiles.json`，后续可在这个文件里调整试玩时长、AI 模式、截图数量和 trace 策略。
- 当前 Windows 打包版默认数据目录示例：如果 exe 放在 E 盘，则使用 `E:\H5游戏评测助手数据`。工具顶部提供“打开数据目录”按钮。

## 当前验证记录

- `poc_review` 已完成 3 款真实批量验证：`pixel-match`、`dead-rush`、`idle-prison-life` 均完成采集、Gemini 评测、本地视频和飞书写入。
- `full_30min` 已完成 1 款正式采样验证：`pixel-match` 总耗时约 34.4 分钟，4 个设备/网络档位全部采集成功，生成 4 张截图、1 段 30 秒本地视频，视频约 2.98 MB，Gemini 和飞书写入均成功。
- v1.0 RC1 前 UI 已用 Playwright 做桌面 `1440x1050`、全屏宽屏和窄屏检查；桌面主流程保持三栏工作台结构，窄屏表格在内部横向滚动。
- v1.1 RC1 将打包到 `dist\H5游戏评测助手 1.1.0-rc.1.exe`；基础配置检查可读取当前数据、4 款游戏和可写的飞书字段状态。
- v1.1 Alpha2 新增“未完成项”回填：批量任务取消或中断后，可以把失败、取消、未启动的游戏 ID 填回队列继续跑。
- v1.1 Alpha3 新增批次历史筛选：可按未完成、失败、已取消、成功、执行/预演和关键词筛选历史批次。
- v1.1 Alpha4 新增批次详情和日志入口：历史批次可查看任务明细，并打开 stdout/stderr 本地日志。
- v1.1 Alpha5 新增细分耗时估算和批次报告导出：可在批量运行前查看耗时拆分，并从历史批次生成 HTML/JSON 报告。
- v1.1 RC1 已通过 `cow-saver` 单款 `full_30min` 正式抽样验收：采集、视频、Gemini、飞书写入和批次报告均成功。

## 当前限制

- 当前 exe 未签名，Windows 可能提示未知发布者。
- 当前已替换为 RC 临时 H5 图标，正式发布前仍可替换为品牌图标。
- 单文件 portable exe 更适合预览；当前已默认使用 exe 所在盘符根目录下的 `H5游戏评测助手数据` 保存配置、证据和运行日志，后续可以再增加“选择工作目录”设置。
- 当前视频证据只保存在本地路径；团队协作时如需共享视频，应另行放到云盘、对象存储或内网文件服务。
