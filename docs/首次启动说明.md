# H5 游戏评测助手首次启动说明

适用版本：v1.3.0-rc.1 及后续正式版。

## 第一次打开

1. 解压 `H5游戏评测助手-v1.3.0-rc.1.zip`。
2. 双击里面的 `H5游戏评测助手 1.3.0-rc.1.exe`。
3. 工具会在当前盘符创建数据目录，例如 `E:\H5游戏评测助手数据`。
4. 点击顶部“打开目录”可以查看配置、证据、报告和运行日志。

如果 Windows 提示“未知发布者”，这是因为当前测试包未做应用商店签名。确认文件来自本项目 GitHub 下载包后再运行。

## 首次必填

打开“配置中心”，按顺序完成：

1. 填写 Gemini API Key，模型可先使用默认值。
2. 填写飞书 App ID 和 App Secret。
3. 粘贴飞书多维表格 URL，或填写 `app_token` / `table_id`。
4. 点击“保存配置”。
5. 点击“快速检查”，确认 Gemini、飞书字段和标签库都可用。

密钥只保存在本机数据目录里。已保存的 API Key 和 App Secret 不会在界面回显。

## 飞书接入

飞书配置步骤比较多，工具内“飞书向导”会按 6 个步骤展示：

1. 创建企业自建应用。
2. 复制 App ID / App Secret。
3. 开通多维表格和 Wiki 相关权限。
4. 发布应用版本，让权限正式生效。
5. 在多维表格里添加文档应用，并设为可编辑。
6. 回到工具检查字段和同步标签库。

更详细的逐步说明见 `docs\飞书接入用户操作指南.md`。

## 运行一款游戏

1. 在“工作台”导入 H5 游戏链接，或在“游戏库”选择已有游戏。
2. 选择运行档位：`poc_review` 用于短测，`full_30min` 用于正式 30 分钟评测。
3. 点击“开始运行”。
4. 运行完成后，在右侧查看质量提示、截图证据、AI 字段快照和人工复核。
5. 需要人工确认时，填写复核备注后保存；飞书配置完整时会同步复核状态。

## 证据文件

每款游戏会自动创建独立文件夹：

```text
数据目录\evidence\<game_id>\
```

截图、视频、报告、AI 结果、飞书写入预览和运行记录都保存在这里。

- 截图默认保存在本地；可在配置中心选择写入飞书时上传截图附件。
- 视频默认只保存在本地，不上传飞书。
- `poc_review` 是短测档；`full_30min` 才是正式 30 分钟评测档。

## 标签库和写回

玩法、题材、画风、特色标签、适合人群、操作方式等选项可以维护在飞书标签库表里。

1. 在“配置中心”同步标签库。
2. AI 会优先从标签库里选择匹配项。
3. 如果 AI 认为现有选项不足，会提示新标签建议。
4. 新标签建议需要人工确认后再写回飞书。

## 常见问题

- 如果 Gemini 失败，先检查 API Key 和网络代理，再点“快速检查”。
- 如果飞书写入失败，优先检查应用是否发布、权限是否开通、文档应用是否已添加到多维表格。
- 如果批量任务中断，可以在“任务队列”查看失败项并继续跑。
- 不要把 Gemini API Key、Feishu App Secret 或本地配置文件发给他人。
