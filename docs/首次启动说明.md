# H5 游戏评测助手首次启动说明

适用版本：v1.2 RC1。

## 第一次打开

1. 双击 `H5游戏评测助手 1.2.0-rc.1.exe`。
2. 工具会在 exe 所在盘符根目录创建数据目录，例如 `E:\H5游戏评测助手数据`。
3. 点击顶部“打开目录”可以查看配置、证据、报告和运行日志。

## 必填配置

1. 打开“配置中心”。
2. 填写 Gemini API Key，模型可先使用默认值。
3. 填写飞书 App ID、App Secret、多维表格链接或 `app_token` / `table_id`。
4. 点击“快速检查”或配置页里的检查按钮，确认 Gemini、飞书字段和标签库都可用。

## 飞书接入

飞书配置步骤比较多，工具内“飞书向导”会按 6 个步骤展示：

1. 创建企业自建应用。
2. 复制 App ID / App Secret。
3. 开通多维表格和 Wiki 相关权限。
4. 发布应用版本，让权限正式生效。
5. 在多维表格里添加文档应用，并设为可编辑。
6. 回到工具检查字段和同步标签库。

更详细的逐步截图说明见 `docs\飞书接入用户操作指南.md`。

## 运行一款游戏

1. 在“工作台”导入 H5 游戏链接，或在“游戏库”选择已有游戏。
2. 选择运行档位：`poc_review` 用于短测，`full_30min` 用于正式 30 分钟评测。
3. 点击“开始运行”。
4. 运行完成后，在右侧查看质量提示、截图证据、AI 字段快照和人工复核。
5. 需要人工确认时，填写复核备注后保存；飞书配置完整时会同步复核状态。

## 证据文件

每款游戏会自动创建独立文件夹：

`数据目录\evidence\<game_id>\`

截图、视频、报告、AI 结果、飞书写入预览和运行记录都保存在这里。视频默认只保存在本地，不上传飞书；截图是否上传飞书由配置开关控制。

## 标签库和写回

v1.2 RC1 增加了标签库管理视图：

1. 在“配置中心”同步标签库。
2. 使用搜索框按中文名、英文名或 option id 查找标签。
3. AI 提出新标签建议后，先人工选择“接受”“暂不加入”或“需补充”。
4. 只有“接受”的建议才会进入写回预览。
5. 点击“写回飞书”会修改 Taxonomy Options 表，请先用 1 条建议小范围验证。

## 当前注意事项

- v1.2 RC1 是候选包，适合个人和小范围朋友测试，不是正式公开分发版。
- exe 未签名，Windows 可能提示未知发布者。
- 不要把 Gemini API Key、Feishu App Secret 或本地配置文件发给他人。
- 如果运行失败，先查看底部“任务日志”，再用“快速检查”确认配置状态；批量任务取消后，可以在任务队列里点击“未完成项”继续跑剩余游戏。
