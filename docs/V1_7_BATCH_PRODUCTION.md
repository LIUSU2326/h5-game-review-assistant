# v1.7 批量生产体验与失败恢复

版本：v1.7.0-alpha.1

## 目标

v1.7 面向更高频的真实批量生产：跑批前能看清范围，跑批后能快速知道是否需要恢复，失败后能一键回填队列，并能从历史中发现反复失败的游戏。

## v1.7.0-alpha.1 已完成

- 服务端 `/api/status` 新增 `batch.production`。
- 生产总览会读取 `batch/runs/*/run.json`，统计最近执行批次窗口。
- 总览包含执行批次数、预演批次数、最近成功率、可恢复游戏数、重试救回数、重复失败游戏和每款游戏任务历史。
- 恢复建议会优先使用最近执行批次的未完成项；没有未完成项时使用失败项。
- 任务队列主区新增“v1.7 批量生产总览”，可一键填入恢复队列。
- 右侧任务队列上下文面板同步展示生产总览，便于查看批次详情时直接恢复。

## 验收重点

- 没有历史批次时，总览显示无历史状态，不阻断预演和运行。
- 最近批次成功时，可恢复数量为 0，并显示无需恢复。
- 最近批次存在失败、取消或未启动任务时，点击恢复按钮会把对应 `game_id` 回填到队列输入框。
- 存在多次失败的游戏时，总览显示失败次数、档位和错误尾巴。
- 队列页在桌面窗口和全屏状态下不出现文字挤压、按钮遮挡或滚动区域丢失。

## 后续计划

- 把生产总览写入批次 HTML 报告。
- 在批次历史筛选中增加“有重复失败”和“被重试救回”筛选。
- 为单款游戏详情补充最近批量任务历史。
- 增加批量恢复前预检：提示会重跑哪些游戏、预计耗时和是否会重新写入飞书。
