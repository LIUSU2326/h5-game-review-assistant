# H5 游戏评测助手版本路线

更新时间：2026-06-02

## 当前版本

- 稳定下载版：v1.6.0。
- 当前开发版：v1.8.2-alpha.2。

## v1.6.0：AI 自动试玩增强正式版

定位：可给内部或朋友试用的本地桌面内测版。

已经具备：

- 导入 H5 游戏链接并维护游戏库。
- 使用 Playwright/Edge 采集多设备截图、弱网页面证据和本地视频。
- 每款游戏独立保存本地证据文件夹。
- 使用 Gemini 生成分类、玩法、人群、英文介绍、How To Play、FAQ 等字段。
- 从飞书多维表格同步标签库，并可将评测结果写回飞书。
- 工作台、任务队列、游戏库、证据复核、配置中心、飞书向导六个主模块。
- v1.3.0-alpha.8 已完成 UI 收口第一轮：全屏自适应、右侧上下文面板精修、只展示状态与可点击按钮区分、中文文案优化。
- v1.3.0-beta.1 已完成 4 款游戏批量烟测、4 款 Gemini 字段完整性检查和飞书写入验证。

## v1.3.0-alpha.8：UI 收口版

目标：让工具在桌面窗口和全屏状态下更稳定、更清晰、更像一个可交付的工作台。

验收重点：

- 全屏后主工作区、游戏库、证据复核列表能够自适应拉伸。
- 左侧导航、右侧上下文面板、底部日志不互相挤压。
- “只展示”的指标、流程节点、状态条不再伪装成按钮。
- 右侧各模块字号、行高、间距和按钮层级更统一。
- 工作台、任务队列、游戏库、证据复核、配置中心、飞书向导切换状态明确。

## v1.3.0-beta.1：功能验收版

目标：从“能跑通”推进到“批量跑得稳”。

计划内容：

- 批量稳定性验收：多游戏连续运行、失败不阻断、失败项可复跑。
- 证据文件夹验收：截图、视频、报告、payload、日志归档完整。
- 飞书写入验收：字段完整、状态准确、失败原因可见。
- Gemini 输出验收：核心分类字段和英文导出字段完整。
- 按钮状态收口：不能使用的入口明确置灰或转成状态展示。

已验收：

- `poc_smoke` 批量烟测 4/4 成功：Cow Saver、Pixel Match、Dead Rush、Idle Prison Life。
- 每款游戏生成 4 张截图，本地证据目录、报告、payload、运行记录完整。
- 4 款游戏均使用 Gemini 生成评测，中文后台字段和英文导出字段必填项无缺失。
- 4 款游戏均成功更新到飞书多维表格，每条记录写入 35 个字段。
- 命令行工具已统一支持 `--key=value` 参数写法，避免误用默认运行档位。

## v1.3.0-rc.1：候选正式版

目标：适合小范围分发试用。

计划内容：

- 修掉 alpha/beta 发现的明显 bug。
- 整理首次启动说明、下载说明、配置向导。
- 确认 portable exe 可在新电脑上解压运行。
- 更新 GitHub 下载包和试用说明。

已完成：

- 首次启动说明更新到 v1.3 口径。
- 新增下载与试用说明。
- 飞书清单从命令行配置口径调整为桌面工具配置中心口径。
- 飞书向导文案改为面向首次配置者的核对说明。

## v1.3.0：正式可用版

目标：本地 H5 游戏评测工具稳定发布。

边界：

- 只覆盖 H5 网页游戏。
- 证据文件默认保存在本地。
- 截图上传飞书是可选项。
- 视频不上传，仅本地保存。
- 不包含小程序、App、云真机、多账号权限系统。

发布状态：

- v1.3.0-beta.1 已完成 4 款样本批量稳定性、Gemini 字段完整性和飞书写入验证。
- v1.3.0-rc.1 已完成首次启动说明、下载说明和配置向导收口。
- v1.3.0 正式包沿用 rc.1 核心链路，并更新正式版下载入口。

## v1.4：证据复核增强

目标：让截图、视频和人工复核更适合团队协作。

计划内容：

- 更好的截图/视频浏览。
- 人工复核状态、备注、复核时间记录。
- 质量问题追踪。
- 本地证据包导出。
- 飞书记录定位到本地证据或截图附件。

v1.4.0-alpha.1 已完成：

- 证据复核面板新增“导出证据包”按钮。
- 工具会将单款游戏证据目录压缩到 `outputs/evidence-packages/`。
- 导出包包含该游戏的截图、报告、AI 结果、payload、复核记录和运行记录。
- 导出过程只保存本地文件，不上传飞书。

v1.4.0-alpha.2 已完成：

- 截图缩略图支持工具内放大预览。
- 预览层支持 Esc 关闭，并保留“打开原图”入口。

## v1.5：字段和标签库增强

目标：让玩法、题材、画风、标签库更适合运营维护。

计划内容：

- 字段映射可视化管理。
- 分类库、玩法库、题材库、画风库同步和检查。
- AI 新标签建议进入待确认列表。
- 飞书模板版本检查。
- 字段缺失、类型不匹配、选项缺失有清晰提示。

v1.5.0-alpha.1 已完成：

- 配置中心新增模板版本审计条。
- 汇总本地字段映射、飞书远端字段和标签库同步状态。
- 生成模板版本号、Mapping SHA256、Remote Schema SHA256 和 Taxonomy SHA256。
- 可复制模板摘要，方便排查飞书表是否和工具字段库一致。

## v1.6：AI 自动试玩增强

目标：让工具更像玩家一样探索游戏。

当前进度：

- v1.6.0-alpha.1 已开始：新增 `adaptive_probe` 策略和配置中心“AI Player”策略面板。
- v1.6.0-alpha.2 已完成：新增运行前预检，开始运行前提示策略、耗时、Gemini/飞书配置和批量风险。
- v1.6.0-alpha.3 已完成：新增自动试玩复盘，跑完后可查看动作质量和重跑建议。
- v1.6.0-beta.1 已完成：整合 v1.6 alpha 功能，发布第一版 AI 自动试玩增强 beta 包。
- v1.6.0-rc.1 已完成：整理下载入口、首次启动说明和候选发布包，准备小范围分发试用。
- v1.6.0 已完成：正式版发布，包含运行前预检、自动试玩复盘和 v1.6 下载包。
- 当前自动试玩仍是 Playwright 启发式操作，不逐帧调用 Gemini，因此不会因为自动试玩本身明显增加 API 消耗。
- 新增 `ai_probe_alpha` 档位，用于在单款或小批量游戏上验证 AI 玩家预留策略。

计划内容：

- AI 玩家模式配置。
- 自动点击、滑动、拖拽、键盘操作增强。
- 新手引导识别。
- 目标、失败、通关、升级、广告弹窗识别。
- 更完整的 AI 行为轨迹日志。

## v1.7：批量生产体验和失败恢复增强

目标：让真实生产批量更容易监控、恢复和复盘，减少“跑完以后才知道哪里断了”的成本。

计划内容：

- 批量生产总览：基于最近批次统计成功率、失败数、重试效果和平均耗时。
- 失败恢复建议：自动判断最近执行批次应该优先填入未完成项还是失败项。
- 任务历史增强：按游戏聚合最近运行状态、失败次数、重试次数和错误尾巴。
- 批量总览增强：在任务队列主区和右侧上下文面板同步展示生产状态。
- 批次报告增强：把恢复建议和重复失败游戏作为后续报告扩展方向。

v1.7.0-alpha.1 已完成：

- 服务端新增批量生产总览数据：`batch.production`。
- 总览包含最近执行批次窗口、成功率、可恢复游戏、重试救回数、重复失败游戏和每款游戏任务历史。
- 任务队列 UI 新增“v1.7 批量生产总览”，可一键把恢复建议填入队列输入框。
- 右侧任务队列上下文面板同步展示生产总览，便于边看批次详情边恢复。
- README 和版本路线更新到 v1.7.0-alpha.1 开发口径，v1.6.0 仍作为稳定下载版。

## v1.8：飞书表结构和标签库闭环

目标：先为后续大功能用户体验测试准备一个更清晰的工具逻辑底座，减少主表字段膨胀，把分类选项、AI 新标签建议和正式评测结果拆开。

当前口径：v1.8.2-alpha.2 已进入飞书标签库和截图附件写入验证线，不继续投入测试链接验证；后续以用户体验反馈和工具逻辑调整为主。

v1.8.0-alpha.2 已完成：
- `Art Style` 改为多选口径，和题材、特色标签一起进入分类闭环优先级。
- 飞书 payload 生成阶段新增标签库预检，检查玩法/题材/画风/特色标签/人群/操作方式是否存在于 `Taxonomy Options`。
- 缺失分类选项会标记为 `taxonomy_review_required`，并阻止直接写入正式结果表，避免污染飞书多选字段。
- 任务队列新增清空队列、移除当前游戏和单项移除入口。
- 主 pipeline 不再生成旧 POC Workbench，保留桌面工具和单游戏 `report.html` 作为主要查看入口。

v1.8.1-alpha.1 已完成：
- 工作台新增“当前下一步”动作按钮，可直接跳转配置、同步标签库、检查字段、导入链接或预演队列。
- 运行按钮继续使用门禁逻辑，缺 AI、缺飞书、字段未检查或未确认时不允许盲目运行。
- 配置右侧状态面板新增“字段和标签库”状态，集中显示字段差异、标签库同步数量和截图写入口径。
- 飞书标签库状态条增加用途说明，明确玩法、题材、画风、特色标签等选项来自飞书。
- Windows 打包白名单补入 `config/field_composer.defaults.json`，避免打包后字段编排默认模板缺失。

v1.8.2-alpha.1 已完成：
- 标签库预检改为优先校验字段编排器里实际会写入飞书的单选/多选字段，拖出输出分类的字段不再阻塞写入。
- 标签库预检新增分类汇总，集中显示玩法、题材、画风、特色标签、人群和操作方式的匹配数与缺失数。
- 缺失选项会补进 `Taxonomy Suggestions`，并继续要求先复核再写入，避免直接污染正式评测字段。
- 截图附件写入状态透传到前端，能区分本地路径、预演跳过、标签阻塞、已准备附件和部分上传失败。
- 配置摘要里的截图口径改为“飞书附件字段 / 本地路径”，避免误解为所有截图都会自动显示在表格里。

v1.8.2-alpha.2 已完成：
- `Suggestion Review` 会读取每款游戏 `feishu_payload_preview.json` 里的标签预检缺失项，避免“预检知道缺标签，但复核列表看不到”的断点。
- 服务端标签分类映射兼容 `Game Type`、`Art Style`、`Target Audience` 等带空格字段名，写回 `Taxonomy Options` 时能正确归类。
- 建议项显示来源和分类，区分 AI 建议与写入预检缺失项。
- 标签建议被接受/拒绝后会清空旧写回预览，要求重新生成预览后才能写回飞书。
- 没有写回预览或预览为空时，`写回飞书` 按钮禁用，减少误操作。

v1.8.2-alpha 后续真实验证：
- 用真实飞书多维表格验证玩法、题材、画风、特色标签等单选/多选字段的选项读取。
- 验证确认后的缺失选项能写回 `Taxonomy Options`，并且多用户共用同一份表格时不会误删已有字段。
- 验证截图附件上传后在飞书里是否能直接查看，并保留本地路径兜底。

v1.8.3-beta 计划：
- 进入大功能用户体验测试，只修真实使用中出现的卡点。
- 根据反馈删减重复入口和无意义提示，不再新增装饰性功能。
- 汇总字段编排、配置路径、批量队列和证据复核的最终交互问题。

计划内容：
- 重新梳理飞书主表字段，让主表只保留游戏结果、核心分类、英文内容和复核状态。
- 使用 `Taxonomy Options` 作为正式标签库表，维护玩法、题材、画风、特色标签、适合人群和操作方式等选项。
- 后续新增 `Taxonomy Suggestions` 标签建议表，用于接收 AI 认为现有列表不足时产生的新分类建议。
- 写入飞书前检查分类字段是否匹配标签库选项，缺失选项先进入建议流程，不直接污染正式字段。
- 确认并修正 `Art Style` 单选/多选口径，优先支持题材、画风、特色标签这些已存在多选列表的字段。
- 队列管理增加清空队列、移除单项等轻量操作，配合批量恢复入口收敛。
- 旧 POC Workbench、raw JSON 预览和重复入口后续降级为高级调试能力。

## v1.9：AI 准确性增强

目标：在 v1.8 表结构和标签闭环稳定后，再集中提升 AI 判断质量和自动试玩有效性。

计划内容：
- 提升 playable URL 识别、iframe/redirect/CDN 链路判断和真实可玩状态识别。
- 优化截图选择和自动试玩路径，减少广告、空页面、加载页对分类结果的干扰。
- 让 AI 分类结果提供证据说明和低置信度提示，便于人工复核。
- 提升玩法、题材、画风、特色标签的匹配准确性。

## v2.0：多平台版

目标：从 H5 网页扩展到小程序和 App。

计划内容：

- 微信小程序接入方案。
- Android 自动化采集。
- iOS 自动化采集。
- 真机或云真机任务执行。
- 多平台字段差异映射。

## 当前建议优先级

1. 以 v1.8.2-alpha.2 为当前飞书标签库验证线，先确认输出字段、单选/多选选项、缺失建议写回和截图附件状态是否足够清楚。
2. v1.8.2-alpha 后续只做真实飞书表格验证，不再新增无关入口。
3. v1.8.3-beta 再做大功能用户体验测试反馈，只修真实卡点，不继续堆功能。
4. v1.9 再集中处理 AI 准确性，v2.0 多平台能力暂缓。
